2.11. pctheory.tables

This module contains functionality for creating tables of set-class names. Every SetClass12 object requires a name table for looking up Forte and Carter names, as well as looking up Z-relations. The name table is a dictionary that contains dictionaries and lists with the data, and is automatically created when you initialize a SetClass12 object. However, for efficiency, you can create the name table independently and pass it as an argument to the SetClass12 conctructor. This allows multiple SetClass12 objects to share the same name table, saving memory and execution time when working with many SetClass12 instances.

Name tables are not part of SetClass24 objects at this time. There are several reasons for this. First of all, Forte‐style names are impractical in the microtonal system because of the huge number of set-classes. So there is no need to look up additional names for a SetClass24 object. Also, a name table for microtonal set-classes would require an enormous amount of memory.

create_tables_set()dict

Creates a dictionary with several sub-lists: hexChars, hexToInt, and several sub-dictionaries: setToForteNameTable, setToForteNameTableLeftPacking, forteToSetNameTable, zNameTable, forteToCarterNameTable, carterDerivedCoreTable.

create_tables_row()dict

Creates a dictionary with one sub-dictionary: allIntervalRowGenerators.
